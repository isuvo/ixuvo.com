# ixuvo.com Personal Website

Astro-powered personal website and blog for `ixuvo.com`. The site runs locally on the home server and is exposed publicly through Cloudflare Tunnel.

## Overview

This project contains:

- Personal profile homepage
- Blog listing and individual blog post pages
- Owner-only write/edit page for blog publishing
- PocketBase-backed content API integration
- Resume download and static assets
- RSS feed and sitemap generation
- Health endpoint for monitoring

Production URL:

```text
https://ixuvo.com
```

Local URL:

```text
http://shuvo-dell:3000
```

## Tech Stack

- Astro
- Node.js adapter for server output
- PocketBase for blog content and author login
- Cloudflare Tunnel for public domain access
- Gatus for monitoring

## Project Location

On the home server, the source code is located at:

```text
C:\HomeServer\apps\site
```

Important folders:

```text
src\pages          Astro routes and pages
src\layouts        Shared page layout
src\lib            PocketBase, GitHub, and profile helpers
public             Static files such as logo, favicon, resume, images
scripts            Utility scripts
dist               Production build output, generated by Astro
node_modules       Installed dependencies
```

## Main Routes

```text
/                  Homepage
/blog              Blog listing
/blog/[slug]       Individual blog post
/write             Owner-only blog editor
/rss.xml           RSS feed
/sitemap-index.xml Sitemap
/healthz           Health check endpoint
```

## Content Source

Blog posts are stored in PocketBase.

Local PocketBase URL:

```text
http://shuvo-dell:8090
```

Public PocketBase URL:

```text
https://pb.ixuvo.com
```

PocketBase is protected through Cloudflare Access on the public domain. Public signup is disabled. Only the owner account is intended to publish content.

## Development

Install dependencies:

```powershell
cd C:\HomeServer\apps\site
npm install
```

Run the local development server:

```powershell
cd C:\HomeServer\apps\site
npm run dev
```

The dev server listens on:

```text
http://shuvo-dell:3000
```

## Build

Build the production site:

```powershell
cd C:\HomeServer\apps\site
npm run build
```

Astro writes the production output to:

```text
C:\HomeServer\apps\site\dist
```

Important: files in `public` are copied into `dist` only during build. If you replace a file such as the resume PDF, run `npm run build` before restarting the service.

## Production Service

The website runs as a Windows service:

```text
HomeServerSite
```

Service wrapper:

```text
C:\HomeServer\ops\site\HomeServer-site.exe
```

Restart the site after a build:

```powershell
& 'C:\HomeServer\ops\site\HomeServer-site.exe' stop
Start-Sleep -Seconds 4
& 'C:\HomeServer\ops\site\HomeServer-site.exe' start
```

Check service status:

```powershell
Get-CimInstance Win32_Service -Filter "Name='HomeServerSite'" |
  Select-Object Name,State,ProcessId,StartMode
```

## Updating The Resume

Resume file:

```text
C:\HomeServer\apps\site\public\isuvo-resume.pdf
```

After replacing the PDF:

```powershell
cd C:\HomeServer\apps\site
npm run build
& 'C:\HomeServer\ops\site\HomeServer-site.exe' stop
Start-Sleep -Seconds 4
& 'C:\HomeServer\ops\site\HomeServer-site.exe' start
```

Then verify:

```text
https://ixuvo.com/isuvo-resume.pdf
```

If the browser still shows the old file, try a hard refresh or use a cache-busting query string:

```text
https://ixuvo.com/isuvo-resume.pdf?fresh=1
```

## Environment

The Astro config uses `SITE_URL` when available:

```text
SITE_URL=https://ixuvo.com
```

If `SITE_URL` is not set, the site falls back to:

```text
http://shuvo-dell:3000
```

Do not commit secrets, passwords, Cloudflare tokens, PocketBase database files, or local service credentials.
